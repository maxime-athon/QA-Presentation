Vision Globale

Qualité : Le Contrat de Confiance

Dans un monde où chaque seconde dépend d'un algorithme, la qualité logicielle est la frontière entre un service d'excellence et un échec critique.

Économie

"Un bug coûte cher."

Plus on découvre une erreur tard, plus sa correction demande de ressources. Le test permet de prévenir plutôt que de subir des pertes financières massives.

Sécurité

"Zéro risque."

Pour des logiciels médicaux, bancaires ou de transport (comme Ariane 5), le bug n'est pas permis. Tester, c'est garantir que le code fait exactement ce qu'on attend de lui.

Réputation

"Fidéliser."

Une application qui "plante" est une application désinstallée. La qualité assure une expérience utilisateur fluide, socle de la réputation de toute entreprise numérique.

En résumé, c'est quoi ?

C'est l'ensemble des méthodes qui permettent de s'assurer qu'un logiciel fonctionne bien, qu'il est sécurisé et qu'il pourra évoluer sans casser dans le futur.

Architecture des Tests

La Pyramide de Qualité

On ne teste pas une vis comme on teste un moteur complet. Pour être efficace, le test doit être stratégique et hiérarchisé.

End-to-End (E2E)
Tests d'Intégration
Tests Unitaires (La Base)

Concept de Mike Cohn : Tester plus à la base, moins au sommet.

01. Tests Unitaires

On teste une seule "unité" de code (une fonction, une classe) de manière totalement isolée.

  • Ultra Rapides
  • Très Précis
  • Coût Faible

02. Tests d'Intégration

On vérifie que les différentes pièces du puzzle communiquent bien entre elles (ex: Le Site <-> La Base de données).

  • Vérifie les Flux
  • Teste les Liaisons

03. Tests Fonctionnels

Le test "métier". Est-ce que le bouton "Ajouter au panier" ajoute bien l'article ? On teste le résultat final.

  • Vision Métier
  • Conformité

04. End-to-End

On simule un vrai utilisateur sur son navigateur du début à la fin de son parcours.

  • Scénario Réel
  • Lents & Coûteux

Ce qu'il faut retenir :

"Plus on monte dans la pyramide, plus les tests sont lents et chers. Une bonne stratégie possède une base solide de tests unitaires pour détecter les erreurs immédiatement."

Étape Suivante : CI/CD
Continuous Integration

L'Automatisation CI/CD

Transformer les tests en une barrière de sécurité automatisée et infranchissable.

Le Pipeline de Qualité

À chaque git push, un robot (Runner) exécute une série d'étapes définies dans un fichier YAML.

1. Build : Compilation du projet
2. Test : Lancement des suites de tests
3. Deploy : Mise en production auto
GitHub Actions
GitLab CI

Avantages Clés

  • Feedback rapide : Erreurs détectées en 2 min.
  • Zéro régression : Le code cassé n'arrive jamais au client.

Le mantra DevOps

"Si ça fait mal, fais-le souvent."
Plus on automatise, plus on réduit la peur de livrer et plus on augmente la qualité globale.

Atelier Pratique

Cas concret : Développement en TDD d'un module de calcul de panier avec Jest.

L'objectif

Nous devons créer une fonction calculateTotal() qui prend une liste d'articles et applique une réduction si le montant dépasse 100$.

Scénarios à tester :

  • Somme de 2 articles sans remise
  • Application d'une remise de 10% (>100$)
  • Gestion d'un panier vide (Cas limite)
cart.test.js
// Étape 1 : RED (Le Test)

test('remise 10% si > 100$', () => {

const items = [{ price: 150 }];

expect(calculateTotal(items)).toBe(135);

});
